# Plandex

## Overview

Open-source CLI-агент, оптимизированный под большие и долгосрочные задачи. Ключевая идея — версионирование всех изменений агента в отдельном буфере с возможностью просмотра и отката до применения в реальный код. Подходит для задач, которые занимают много шагов и файлов.

- **Вендор:** Plandex Inc (open-source)
- **Модель:** GPT-4o, Claude Sonnet (через OpenRouter или напрямую)
- **Open Source:** ✓ AGPL-3.0
- **Сайт:** plandex.ai

## Key Features

- **Versioned changes** — все изменения накапливаются в staging-буфере, применяются одной командой
- **Rollback** — откат к любому предыдущему состоянию изменений
- **Context management** — явное управление контекстом: добавляешь файлы/URL/команды вручную
- **Long tasks** — агент продолжает работу между сессиями, помнит незавершённые задачи
- **Multi-file** — уверенно работает с изменениями в десятках файлов одновременно
- **Self-hosted** — можно развернуть собственный сервер Plandex

## Use Cases

- Большие рефакторинги, затрагивающие всю кодовую базу
- Долгосрочные задачи, которые невозможно решить за одну сессию
- Команды, которым важен review всех изменений до их применения
- Генерация фич "с нуля" по подробному ТЗ

## Setup

```bash
curl -sL https://plandex.ai/install.sh | bash
plandex new
```

## Considerations

- Требует явного управления контекстом — менее автономный чем Claude Code
- UX staging-буфера непривычен и требует привыкания
- Меньше инструментов автоматизации (нет hooks, MCP)
- Облачная версия платная, self-hosted требует настройки
