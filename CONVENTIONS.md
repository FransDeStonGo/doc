# CONVENTIONS — Стандарты базы знаний

Этот документ описывает правила работы с базой знаний: как создавать документы, как их структурировать, как находить нужное. Читать перед добавлением нового контента.

---

## Типология документов

В базе пять типов документов. Тип указывается в frontmatter (`type:`).

| Тип | Назначение | Пример |
|-----|-----------|--------|
| `nav` | Навигационный индекс раздела | `agents/agents.md` |
| `concept` | Объяснение концепции или архитектурного паттерна | `agents/tool-use.md` |
| `reference` | Справочник: модели, инструменты, конфиги | `ai/anthropic/claude/claude.md` |
| `comparison` | Сравнительный анализ нескольких объектов | `ai/cli-agents/cli-agents.md` |
| `guide` | Практическая инструкция how-to | `mcp/setup-custom-server.md` |

---

## Frontmatter

Каждый документ (кроме `project.md` и `INDEX.md`) начинается с YAML frontmatter.

```yaml
---
type: nav|concept|reference|comparison|guide
tags: [agents, tool-use, memory]
status: stub|draft|complete
updated: 2026-03-24
related:
  - agents/memory.md
  - mcp/mcp.md
---
```

**Обязательные поля:**

| Поле | Значения | Описание |
|------|---------|---------|
| `type` | `nav`, `concept`, `reference`, `comparison`, `guide` | Тип документа |
| `tags` | список строк | Ключевые слова для поиска и фильтрации |
| `status` | `stub`, `draft`, `complete` | Степень готовности |
| `updated` | `YYYY-MM-DD` | Дата последнего обновления |

**Опциональные поля:**

| Поле | Описание |
|------|---------|
| `related` | Список путей к связанным документам (относительно `/root/doc/`) |

**Значения `status`:**
- `stub` — только структура, контент не заполнен. Не цитировать как источник.
- `draft` — контент есть, но неполный или требует проверки
- `complete` — актуальный и проверенный контент

## Шаблоны документов

### nav — навигационный индекс

```markdown
---
type: nav
tags: [...]
status: complete
updated: YYYY-MM-DD
---

# [Раздел]

## Overview
Одна-две строки: о чём раздел и зачем.

## Structure
\```
folder/
├── folder.md           # Этот файл
├── subtopic.md         # Описание
└── subdir/             # Описание
    └── subdir.md
\```

## Topics
- **[Тема]** — краткое описание
- **[Тема]** — краткое описание

## Quick Reference (опционально)
Таблица сравнения если нужна.
```

### concept — концепция / паттерн

```markdown
---
type: concept
tags: [...]
status: draft
updated: YYYY-MM-DD
related:
  - path/to/related.md
---

# [Концепция]

## Overview
Что это, зачем нужно, где применяется. 3–5 строк.

## How It Works
Как работает механизм. Схемы текстом, примеры.

## Patterns
Конкретные паттерны реализации с примерами.

## Examples
\```python
# код
\```

## Tradeoffs
Плюсы, минусы, когда применять, когда не применять.

## See Also
- [Название](../path/to/doc.md)
```

## Правила именования

- **Папки:** `lowercase-hyphen`
- **Файлы:** `lowercase-hyphen.md`
- **Навигационный файл раздела:** всегда `<folder-name>.md`
- **Тематические файлы:** по названию темы, не по типу — `tool-use.md`, не `concept-tool-use.md`
- Уникальные имена внутри папки

---

## Правила контента

- **Язык:** русский для описаний и объяснений, английский для технических терминов, кода, имён собственных
- **Глубина:** "для себя через год" — понятно без дополнительного контекста
- **Размер:** предпочтительно 40–150 строк на файл; больше — если тема требует
- **Примеры кода:** всегда с тегом языка (` ```bash `, ` ```json `, ` ```python `)
- **Таблицы:** заголовки столбцов на одном языке (не смешивать русский и английский в одной таблице)
- **Даты:** формат `YYYY-MM-DD`

---

## Перекрёстные ссылки

- Ссылки **относительные** от корня `/root/doc/`: `agents/memory.md`, `../mcp/mcp.md`
- Секция `## See Also` — в каждом `concept`, `reference`, `guide`, `comparison`
- Поле `related:` во frontmatter дублирует See Also для машиночитаемости
- В `nav`-файлах — ссылки на конкретные документы раздела

---

## Обновление INDEX.md

При добавлении нового файла — добавить строку в `INDEX.md`. Формат:

```
| path/to/file.md | type | status | tags | Краткое описание |
```

---

## Инструкция для AI-архивариуса

Если ты AI-агент, работающий с этой базой знаний:

1. **Начинай с `INDEX.md`** — там плоский список всех документов с типом, статусом и описанием. Это быстрее, чем обходить дерево файлов.

2. **Фильтруй по frontmatter:**
   - `type:` — чтобы найти документы нужного типа (только `concept` для паттернов, только `reference` для справочников)
   - `tags:` — для тематической фильтрации
   - `status: stub` — неполные документы, не цитировать как источник факта

3. **Структура каждого документа предсказуема:**
   - `## Overview` — всегда краткое резюме; читай первым для понимания о чём документ
   - `## See Also` + поле `related:` — смежные темы для углублённого исследования
   - `## Tradeoffs` / `## Considerations` — ограничения и нюансы

4. **Навигационные файлы (`type: nav`)** — не содержат контент, только структуру раздела. Используй их для понимания что есть в разделе.

5. **При неопределённости** — читай `CONVENTIONS.md` (этот файл) и `project.md` для понимания целей базы.
