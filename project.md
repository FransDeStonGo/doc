# AI Agent Knowledge Base

## Overview

Личная база знаний по архитектуре, фичам и принципам построения AI-агентов. Цель — сформировать чёткое понимание того, каким должен быть идеальный AI-агент: какие функции ему нужны, как он должен себя вести, какие инструменты использовать.

Документы создаются по мотивам исследований, диалогов и практического опыта работы с современными AI-агентами.

## Structure

```
/root/doc/
├── project.md          # Этот файл — описание проекта и навигация
├── agents/             # Архитектура агентов, паттерны, типы
│   └── agents.md
├── ai/                 # Модели по производителям: Anthropic, OpenAI, Google и др.
│   ├── ai.md
│   └── <vendor>/
├── mcp/                # Model Context Protocol — серверы, интеграции, схемы
│   └── mcp.md
├── providers/          # Провайдеры доступа к API: OpenRouter, Anthropic, OpenAI и др.
│   └── providers.md
└── skills/             # Skills (slash-команды) — создание, примеры, шаблоны
    └── skills.md
```

## Sections

| Папка | Содержание |
|-------|------------|
| `agents/` | Архитектурные паттерны агентов: tool use, memory, планирование, multi-agent системы |
| `ai/` | Модели по производителям: сравнения, характеристики, use cases для агентов |
| `mcp/` | MCP-серверы: протокол, настройка, примеры интеграций |
| `providers/` | Провайдеры API: OpenRouter (приоритет), Anthropic, OpenAI, Google, Groq и др. |
| `skills/` | Кастомные slash-команды: шаблоны, примеры, как писать свои |

## Principles

- Каждая папка имеет свой `<name>.md` — точку входа с описанием структуры
- Документы пишутся по принципу "для себя через год" — понятно без контекста
- Структура плоская там, где достаточно плоской; вложенность только при необходимости
